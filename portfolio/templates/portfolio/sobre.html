Sobre esta Aplicação

1. Arquitetura MVT do Django

O Django usa o padrão MVT — Model, View, Template. O Model define as entidades e comunica com a base de dados. A View recebe o pedido HTTP, consulta o Model e prepara os dados. O Template recebe esses dados e gera o HTML devolvido ao browser.

Nesta aplicação, quando o utilizador acede a /projetos/, a View consulta todos os objetos Projeto na base de dados e passa-os ao template projetos.html, que os apresenta em HTML.

Video-tutorial — Formulário para criar um novo Projeto

2. Modelação

A base de dados tem 9 entidades: Licenciatura, Docente, UnidadeCurricular, Projeto, Tecnologia, TFC, Competencia, Formacao e MakingOf.

Definição em models.py

class Projeto(models.Model):
    titulo = models.CharField(max_length=200)
    descricao = models.TextField()
    classificacao = models.DecimalField(max_digits=4, decimal_places=2)
    destaque = models.BooleanField(default=False)
    unidade_curricular = models.ForeignKey(UnidadeCurricular, on_delete=models.CASCADE)
    tecnologias = models.ManyToManyField(Tecnologia, blank=True)

class Tecnologia(models.Model):
    nome = models.CharField(max_length=100)
    tipo = models.CharField(max_length=20, choices=TIPO_CHOICES)
    descricao = models.TextField()
    nivel_interesse = models.IntegerField(validators=[MinValueValidator(1), MaxValueValidator(5)])
    em_uso = models.BooleanField(default=True)
